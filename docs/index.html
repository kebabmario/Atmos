Introduction

A clean, minimalist weather web app powered by open data.

Atmos is a beautifully designed, minimalist weather web application that delivers real-time weather information with zero setup friction. Built with pure HTML, CSS, and JavaScript ‚Äî no build tools, no frameworks, no API keys required.

Atmos is powered by the Open-Meteo API, a completely free and open-source weather service that requires no registration. Weather icons are rendered using animated SVG files from the Meteocons library by basmilius/weather-icons.

The interface uses a glassmorphism design language ‚Äî frosted-glass cards floating over deep-blue gradient backgrounds ‚Äî with smooth animations and a clean typographic hierarchy.

Getting Started

Up and running in seconds ‚Äî no build step needed.

Clone the repository

git clone https://github.com/kebabmario/atmos.git
cd atmos

Open directly in a browser

The simplest way ‚Äî just double-click index.html or open it via your browser's file menu. No server required.

Serve with a local dev server

For a more realistic experience (especially for geolocation), serve with a local HTTP server:

# Using Node.js (npx)
npx serve .

# Using Python 3
python -m http.server

# Using Python 2
python -m SimpleHTTPServer

Then open http://localhost:3000 (or the port shown) in your browser.

Data Flow on Load

  1. Browser loads index.html which pulls in style.css and app.js.
  2. app.js calls the browser's Geolocation API to get the user's coordinates.
  3. Coordinates are sent to the Open-Meteo forecast API to fetch current, hourly, and 7-day weather data.
  4. The UI is populated: location name (via reverse geocoding), temperature, condition, hourly strip, weekly forecast, and detail cards.
  5. The <body> class is updated to apply the matching dynamic background gradient.

City Search Flow

  1. User types a city name and clicks Go (or presses Enter).
  2. app.js calls the Open-Meteo Geocoding API with the city name.
  3. The first result's latitude/longitude is extracted.
  4. The full weather fetch is triggered with the new coordinates and the UI re-renders.

API Reference

All external API endpoints used by Atmos.

Geocoding API

Converts a city name into geographic coordinates (latitude & longitude).

GET https://geocoding-api.open-meteo.com/v1/search?name={city}&count=1
Parameter Type Description
name string City name to search for
count integer Number of results to return (Atmos uses 1) 
// Example response
{
  "results": [
    {
      "name": "London",
      "latitude": 51.50853,
      "longitude": -0.12574,
      "country": "United Kingdom"
    }
  ]
}

Weather Forecast API

Returns current conditions, hourly data, and 7-day daily forecasts for a given location.

GET https://api.open-meteo.com/v1/forecast
Parameter Value / Example Description

longitude -0.1257 Location longitude

daily weathercode,temperature_2m_max,
temperature_2m_min,sunrise,sunset		 Daily forecast fields (7 days)
hourly temperature_2m,weathercode Hourly forecast fields
timezone auto Auto-detect timezone from coordinates

Features

Everything Atmos shows you at a glance.

Current Weather

The hero card at the top of the left column shows the location name, current weather condition label, temperature in large font (¬∞C by default), and the day's high/low range.

Hourly Forecast

A horizontally-scrollable strip beneath the hero card displays the next 24 hours of forecast, each showing a small animated icon and the expected temperature. Scroll with mouse wheel or swipe on touch devices.

7-Day Forecast

A weekly list below the hourly strip shows the next 7 days: day name abbreviation, animated weather icon, high temperature, and low temperature.

Weather Details Grid

Six glassmorphism detail cards on the right column display:

  • Humidity ‚Äî relative humidity percentage
  • Wind Speed ‚Äî wind speed in km/h
  • Sunrise ‚Äî local sunrise time (HH:MM)
  • Sunset ‚Äî local sunset time (HH:MM)
  • Feels Like ‚Äî apparent temperature in ¬∞C
  • Cloud Cover ‚Äî cloud coverage percentage

The <body> element receives a CSS class based on the current weather condition, triggering a matching gradient background. See the UI & Theming section for the full class list.

UI & Theming

Glassmorphism design language and dynamic weather gradients.

Every card in Atmos uses the following CSS pattern:

Weather icons are served from the Meteocons CDN (basmilius/weather-icons). They are animated SVG files that loop continuously, adding life to the UI without any JavaScript.

Responsive Design

On screens wider than 860px, the app uses a two-column CSS Grid layout:

Configuration & Customization

Tweak Atmos to suit your needs.

Change Default Location

By default, Atmos uses the browser's Geolocation API. To set a static fallback location, find the geolocation error handler in src/app.js and replace the hardcoded fallback coordinates: 

// In src/app.js ‚Äî geolocation fallback
navigator.geolocation.getCurrentPosition(
  (pos) => fetchWeather(pos.coords.latitude, pos.coords.longitude),
  () => fetchWeather(51.5085, -0.1257) // ‚Üê change these defaults (London)
);

Switch to Fahrenheit

Atmos uses Celsius by default. To switch to Fahrenheit, add the temperature_unit=fahrenheit parameter to the Open-Meteo API call in src/app.js: 

// Add &temperature_unit=fahrenheit to the forecast URL
const url = `https://api.open-meteo.com/v1/forecast?latitude=${lat}&longitude=${lon}
  ¤t=temperature_2m,weathercode,...
  &temperature_unit=fahrenheit   ‚Üê add this
  &timezone=auto`;

Add New WMO Codes

To support additional WMO weather codes or override existing labels/icons, edit the WMO_CODES object at the top of src/app.js: 

// In src/app.js
const WMO_CODES = {
  0:  { label: 'Clear Sky',      icon: 'clear-day.svg' },
  1:  { label: 'Mainly Clear',   icon: 'mostly-clear-day.svg' },
  2:  { label: 'Partly Cloudy',  icon: 'partly-cloudy-day.svg' },
  // ... add new codes here:
  100: { label: 'Custom Code', icon: 'your-icon.svg' },
};

Place custom icon SVG files in src/assets/ and reference them by filename.
